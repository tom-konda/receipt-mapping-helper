# 実装共通ルール

全てのサブエージェント・実装者が守るべきルールです。

## ソースコードへのコメント
プロジェクトに配属された新人エンジニアが理解できるよう、コードの意図が自明でない箇所には日本語コメントを残すこと。コメントすべき箇所の例：
- なぜその API やオプションを選択したのか（設計判断の理由）
- ブラウザ互換性やモバイル対応のための回避策
- メモリ管理やリソース解放の意図
- コードから読み取れない業務的な背景

コメント不要な箇所：
- 変数名や関数名から明らかに分かること
- Svelte の基本的な構文
- import 文

## レビュー依頼

実装が終わったら、シニアエンジニアのサブエージェントにコードレビューを受けること
